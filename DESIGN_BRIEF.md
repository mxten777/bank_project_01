# LocalBank ONE 디자인 개선 의뢰서

## 📋 프로젝트 브리핑

### 프로젝트 개요
- **서비스명**: LocalBank ONE
- **유형**: 금융 SaaS 플랫폼 (화이트라벨)
- **타겟 고객**: 농협, 신협 등 지역 금융기관
- **핵심 가치**: 브랜드 즉시 전환 기능 (농협 녹색 ↔ 신협 블루)

### 현재 상태
- ✅ **기능 개발**: 100% 완성 (Firebase 연동 완료)
- ✅ **UI/UX**: 프리미엄급 적용 (Tailwind CSS 기반)
- ✅ **디자인 시스템**: 기본 컬러/애니메이션 구현 완료
- 🎨 **개선 필요**: 브랜드 아이덴티티 강화 및 시각적 세련화

### 디자인 개선 목표
1. 금융권 신뢰감 극대화
2. 브랜드 아이덴티티 강화
3. 사용자 경험 세련화
4. 화이트라벨 전환 시각화

## 🎯 구체적 디자인 요청 사항

### A. 브랜드 아이덴티티 (최우선 순위)

#### 1. 로고 디자인 (2종 제작)
**농협 버전**
- 컬러: 녹색 계열 (#10b981 기반)
- 키워드: 신뢰, 안정, 지역밀착, 따뜻함
- 톤앤매너: 공공성과 안정성을 강조하는 공식적인 톤

**신협 버전**
- 컬러: 블루 계열 (#3b82f6 기반)
- 키워드: 협동, 상생, 혁신, 친근함
- 톤앤매너: 친근하고 일상적인 소통 중심의 톤

**납품 형식**
- SVG 파일 (편집 가능)
- PNG 파일 (투명 배경, 2x/3x 해상도)
- 헤더용: 200×60px (가로형)
- 파비콘: 32×32px, 512×512px (정사각형)
- 다크 모드 버전 (선택사항)

#### 2. 컬러 시스템 최적화
**현재 컬러 (개선 필요)**
```
농협: primary #10b981, secondary #059669, accent #34d399
신협: primary #3b82f6, secondary #2563eb, accent #60a5fa
```

**요청 사항**
- 각 브랜드별 5~7단계 컬러 팔레트 제안
- 접근성 준수 (WCAG AA 레벨, 4.5:1 대비)
- 그라데이션 방향성 가이드
- Error/Warning/Success 상태 컬러
- Neutral 그레이 스케일 (8단계)

**납품 형식**
- HEX, RGB, HSL 코드
- Tailwind CSS 변수명 매칭

#### 3. 타이포그래피 시스템
**폰트 추천**
- 본문용 한글 웹폰트 (Pretendard, Noto Sans KR 등)
- 숫자/영문용 폰트 (선택사항)
- 폰트 라이선스 확인 필수

**타입 스케일**
- H1~H6 제목 크기 (px, rem)
- Body (대/중/소)
- Caption/Label
- 행간(line-height), 자간(letter-spacing)
- 모바일/태블릿/데스크톱 반응형 크기

### B. UI 컴포넌트 디자인 고도화

#### 1. 카드 디자인 시스템
**개선 필요 컴포넌트**
- 공지사항 카드 (목록/상세)
- 문의 카드 (상태별 스타일)
- 대시보드 통계 카드
- 서비스 소개 카드

**디자인 요소**
- 그림자(shadow) 레벨 정의 (sm/md/lg/xl)
- 테두리(border) 스타일 및 반경
- 호버(hover) 상태 효과
- 포커스(focus) 상태 표시
- 액티브(active) 상태

#### 2. 버튼 디자인 시스템
**버튼 타입**
- Primary: 주요 액션 (CTA)
- Secondary: 보조 액션
- Ghost: 투명 배경 버튼
- Danger: 삭제/경고 액션
- Link: 텍스트 링크 스타일

**버튼 크기**
- Large (lg): 48px 높이
- Medium (md): 40px 높이
- Small (sm): 32px 높이

**버튼 상태**
- Default (기본)
- Hover (호버)
- Active (클릭)
- Disabled (비활성)
- Loading (로딩 중)

#### 3. 폼 요소 디자인
**입력 필드**
- Text Input
- Textarea
- Select Dropdown
- Checkbox
- Radio Button

**폼 상태**
- Default (기본)
- Focus (포커스)
- Error (에러)
- Success (성공)
- Disabled (비활성)

**추가 요소**
- Placeholder 텍스트 스타일
- Label 위치 및 스타일
- Helper Text
- Error Message
- Required 표시 (*)

#### 4. 기타 컴포넌트
- Badge/Tag (상태 표시)
- Toast/Alert (알림 메시지)
- Modal/Dialog (팝업)
- Dropdown Menu
- Pagination
- Breadcrumb
- Tabs

### C. 비주얼 에셋 제작

#### 1. 일러스트레이션 (3~5개)
**필수 일러스트**
1. **홈 히어로 섹션** 메인 이미지
   - 크기: 1200×600px (2x 제작)
   - 컨셉: 금융 서비스의 디지털 혁신
   - 스타일: 모던하고 신뢰감 있는 플랫 디자인

2. **404 페이지** 에러 일러스트
   - 크기: 400×400px
   - 컨셉: 길을 잃은 느낌, 친근한 표현
   - 브랜드별 컬러 적용

3. **빈 상태(Empty State)** 일러스트 (2종)
   - 공지사항 없음
   - 문의사항 없음
   - 크기: 200×200px

**선택 일러스트**
4. 서비스 페이지 각 섹션 아이콘 일러스트
5. About 페이지 타임라인 일러스트

**납품 형식**
- SVG (편집 가능)
- PNG (2x, 3x 해상도)

#### 2. 아이콘 세트 (20~30개)
**카테고리별 아이콘**

**금융 서비스** (8개)
- 예금/적금
- 대출
- 체크카드/신용카드
- 보험
- 펀드/투자
- 외환
- 모바일뱅킹
- 인터넷뱅킹

**시스템** (8개)
- 홈
- 검색
- 알림
- 설정
- 사용자
- 로그아웃
- 업로드
- 다운로드

**액션** (8개)
- 추가
- 삭제
- 편집
- 저장
- 취소
- 확인
- 새로고침
- 필터

**상태** (6개)
- 성공
- 에러
- 경고
- 정보
- 로딩
- 체크

**납품 형식**
- SVG 파일 (개별 파일)
- 아이콘 폰트 (선택사항)
- 크기: 24×24px (기본), 확대 가능
- Stroke 또는 Fill 스타일

#### 3. 배경 패턴/텍스처
**필요 패턴**
1. 히어로 섹션 배경 패턴 (subtle)
2. 로딩 상태 배경
3. 푸터 배경 패턴

**요구사항**
- Seamless 타일링 가능
- 브랜드 컬러 적용
- 투명도 조절 가능 (opacity: 0.05~0.1)

### D. 애니메이션 & 인터랙션 가이드

#### 1. 마이크로 인터랙션
**버튼 클릭 효과**
- Scale transform (0.95~1.0)
- 그림자 변화
- 컬러 변화 타이밍

**카드 호버 효과**
- Lift 애니메이션 (translateY)
- 그림자 증가
- 테두리 컬러 변화

**페이지 전환**
- Fade In/Out
- Slide 방향 (좌우, 상하)
- Duration 추천

#### 2. 로딩 애니메이션
**브랜드 특화 스피너**
- 농협/신협 각 1종
- 컬러 적용
- 크기별 제공 (sm/md/lg/xl)

**스켈레톤 UI**
- 카드 로딩 스켈레톤
- 리스트 로딩 스켈레톤
- Wave 애니메이션 효과

#### 3. 트랜지션 타이밍
**Easing 함수 추천**
- 기본: ease-in-out
- 빠른 액션: ease-out
- 드롭다운: ease-in

**Duration 가이드**
- 빠름: 150ms (호버)
- 중간: 300ms (페이지 전환)
- 느림: 500ms (복잡한 애니메이션)

**Phase 4: 가이드 문서화 (2~3일)**
- Style Guide 작성
- 개발 전달 자료
- 최종 검수

**전문 에이전시**: 600만원~1,000만원

**포함 사항**
- 로고 디자인 (2종)
- 디자인 시스템 구축
- 비주얼 에셋 제작
- 가이드 문서
- 2~3회 수정

**별도 비용**
- 추가 일러스트레이션
- 애니메이션 제작 (Lottie 등)

   - 주 2회 진행 상황 공유
   - 초기 컨셉은 빠른 피드백 필수

3. **우선순위 명확화**
   - 로고 > 컬러 시스템 > 컴포넌트 > 에셋 순서

4. **기술적 제약 사항 공유**
   - Tailwind CSS 기반임을 강조
   - 브랜드 전환 시 자동 적용 필요

### 개발 연동 고려사항
1. **Figma 컴포넌트 → React 컴포넌트**
   - Props 구조 유사하게 설계
   - 변수명 일관성 유지

2. **컬러는 CSS 변수로**
   - `--color-primary-500` 형태
   - 브랜드 전환 시 교체 가능하게

3. **SVG 최적화**
   - 불필요한 path 제거
   - viewBox 정확히 설정
   - fill="currentColor" 활용

4. **반응형 기준점**
   - Mobile: < 640px
   - Tablet: 640px ~ 1024px
   - Desktop: > 1024px
